# Jarvis-Life 项目核心亮点介绍

## 1. 一句话定位

**Jarvis-Life 是一个个人生活操作系统：它不只是回答问题，而是持续理解一个人的日程、目标、心理状态、长期偏好和现实环境，并把建议真正落到日程、计划、关怀、记忆和后续反馈里。**

如果传统 AI 助手是一个聊天窗口，Jarvis-Life 更像一个会理解人的生活中枢：它能感知状态、组织多智能体协作、安排执行、沉淀记忆，并在下一次继续接住这个人。

---

## 2. 为什么它不是普通 AI 助手

我们刻意不把 Jarvis-Life 定义为“AI 助手”。助手通常意味着被动响应，而 Jarvis-Life 的核心是主动组织个人生活中的状态、决策和执行闭环。

| 普通桌面助手 | Jarvis-Life |
| --- | --- |
| 以聊天和问答为中心 | 以个人生活决策与执行闭环为中心 |
| 每次对话相对独立 | 有长期记忆、偏好学习和个人画像 |
| 主要给文字建议 | 能生成待确认动作、写入日程、拆解长期计划 |
| 只看用户当前输入 | 同时考虑日程压力、心理状态、时间、天气、位置和偏好 |
| 单模型单视角 | 多智能体角色分工 + 场景圆桌协作 |
| 提醒多是固定规则 | 关怀模块结合情绪、行为、压力趋势和用户反馈 |
| 追求“帮你做事” | 更关注“你现在是否适合做、应该怎么安排、后续如何承接” |

Jarvis-Life 的差异不在于多了几个功能按钮，而在于它试图把人的生活状态、目标执行和长期理解放进同一个系统。

---

## 3. 核心问题：个人生活不是单点任务

真实生活里的问题通常不是一句话能解决的单点任务，而是多个因素互相牵制：

- 日程很多，但不知道怎么取舍；
- 目标很大，但缺少每日执行路径；
- 心理压力、睡眠、体力会影响执行，但传统工具无法感知；
- 附近活动、天气、空档、恢复需求没有被统一考虑；
- AI 工具常常只给建议，不进入日历、任务、记忆和后续提醒；
- 每次对话都像第一次见面，缺少长期偏好和连续上下文。

Jarvis-Life 要解决的是：**把个人生活中的“想法、状态、计划、行动、反馈、记忆”连接成闭环。**

---

## 4. 系统闭环：状态 → 决策 → 执行 → 反馈 → 记忆

Jarvis-Life 的核心流程可以概括为：

```text
用户表达目标 / 状态 / 困扰
  → 系统识别意图、时间、约束和生活上下文
  → 专业智能体或圆桌参与判断
  → 生成可执行计划、待确认动作或关怀建议
  → 写入日程、后台任务、长期记忆或主动消息
  → 用户确认、反馈、调整
  → Shadow Learner 和长期记忆更新个人画像
  → 下一次决策更贴合用户
```

这个闭环让 AI 不再停留在“你可以考虑……”的层面，而是进入“如何安排、谁来判断、何时执行、如何反馈”的系统层面。

---

## 5. 六个差异化创新点

### 5.1 从聊天助手到个人生活操作系统

Jarvis-Life 的目标不是把大模型放进桌面窗口，而是构建一个能承接个人生活连续性的操作系统。用户的日程、计划、心理状态、偏好和现实环境不再分散在不同工具中，而是进入同一个 AI 决策闭环。

普通 AI 工具的输出通常是一段建议；Jarvis-Life 的输出可以是日历事件、后台任务、长期计划、关怀记录、个人画像和下一次决策上下文。

### 5.2 人的状态参与决策

传统效率工具默认人是稳定机器，只要有空档就能安排任务。Jarvis-Life 把心理状态、压力趋势、睡眠节律、日程密度和体力恢复纳入决策。

它不只问“这件事排不排得进去”，还会判断：

- 用户现在是否疲惫；
- 压力是否已经偏高；
- 任务是否应该拆小；
- 是否需要先恢复再执行；
- 晚上是否不适合安排高强度任务。

这使 Jarvis-Life 不只是效率工具，也是一套更人性化的生活决策系统。

### 5.3 多智能体不是角色扮演，而是分工决策

Jarvis-Life 的多智能体不是为了制造聊天热闹，而是为了让复杂生活问题进入不同专业视角。

- Alfred：总管，负责综合统筹和最终协调；
- Maxwell：秘书，负责日程、任务、长期计划和执行路径；
- Mira：心理关怀，负责情绪陪伴、压力识别和安全边界；
- Nora：营养与体力，负责饮食、咖啡因、恢复和能量节奏；
- Leo：本地生活，负责附近活动、恢复场景和生活方式建议；
- Athena：学习策略，负责学习计划、考试准备和认知负荷；
- Shadow：旁路偏好学习器，负责沉淀长期偏好和个人画像。

一个“我今天学不动了但明天要考试”的问题，会同时涉及学习策略、心理压力、日程重排和恢复方案。这不是单一助手视角能稳定处理的。

### 5.4 场景圆桌：复杂生活决策的协作协议

圆桌模块把复杂问题从单轮问答升级为场景化决策流程。不同智能体围绕同一个问题给出观点、风险、分歧和下一步建议，最终沉淀成结构化结果。

它适合处理：

- 今天是否继续学习；
- 周末应该恢复还是外出；
- 日程过载时哪些任务该保留；
- 长期目标如何拆解；
- 心理状态和执行压力冲突时怎么取舍。

圆桌不是“多个角色聊天”，而是个人生活中的多视角决策机制。

### 5.5 行动闭环：建议不是终点

Jarvis-Life 不把 AI 回复当作终点。系统可以把建议转化为：

- 待确认日程；
- 日历事件；
- 长期计划；
- 后台任务；
- 心理关怀记录；
- 主动消息；
- 长期记忆。

用户保留最终确认权，但 AI 不再停留在“你可以考虑一下”的层面。它能把建议推进到真实系统状态里。

### 5.6 Shadow Learner：旁路学习与可见个人画像

Shadow Learner 是 Jarvis-Life 的旁路学习机制。它不直接和用户对话，而是在后台从多次互动中保守提取稳定偏好，例如：

- 用户不喜欢太频繁提醒；
- 上午更适合深度工作；
- 压力大时更需要拆步骤，而不是大道理；
- 晚上不适合高强度任务；
- 用户偏好简短、直接、最多三个选项的建议。

这些信息会进入个人画像和长期记忆，让系统越用越像是理解这个人，而不是每次重新认识用户。

## 6. 核心功能版图

### 6.1 私聊与真实执行步骤

用户可以和不同专业智能体私聊。对话过程中，前端会展示后端真实执行步骤和真实耗时，例如：确认智能体、读取上下文、检索长期记忆、判断工具、生成回复、保存记录。

这让系统从“黑箱等待回复”变成“可观察的执行过程”。

### 6.2 日程与长期计划

系统支持从自然语言创建日程、更新日程、删除日程、查找空档、拆解长期计划、生成每日计划，并将计划日投射到日历中。

近期修正后的日程去重规则更贴合真实使用：同一天同名日程会拦截，不同天同名日程允许存在，适合周期性计划。

### 6.3 心理关怀模块

Mira 可以识别压力和疲惫表达，生成轻量陪伴、关怀建议和状态记录。系统支持关怀趋势、每日详情、主动消息和用户反馈，避免过度打扰。

它不是医疗诊断，而是轻量心理支持、状态记录和生活节奏调节。

### 6.4 多智能体圆桌

圆桌提供 Decision 和 Brainstorm 两类场景：

- Decision：用于复杂取舍和最终建议；
- Brainstorm：用于发散创意、沉淀想法、识别张力和下一步。

前端保留“会议舞台感”，让角色逐个发言并形成主持总结。

### 6.5 本地生活与外部检索

系统具备本地生活搜索、天气、新闻、网页搜索等工具能力。Leo 可以把附近活动、天气、空档和用户状态结合起来，帮助用户选择更合适的恢复方式。

当前更适合演示“本地生活机会雷达”和“外部资料辅助决策”，品牌商品新品检索可作为后续专门工具扩展。

### 6.6 长期记忆与个人画像

系统会将稳定偏好、约束、历史对话和长期记忆沉淀为个人画像。用户可以在前端查看画像和长期记忆摘要，后续对话可以使用这些信息。

## 7. 技术实现亮点

### 7.1 前后端解耦

前端页面不直接拼后端接口，而是通过 `shadowlink-web/src/services/*.ts` 作为接口层。日程、关怀、记忆、设置、圆桌等模块逐步形成边界，便于并行开发。

### 7.2 SSE 流式执行过程

私聊接口支持 `chat_step` 流式事件，步骤包含 `running / done` 状态、真实耗时和细节。前端按步骤 ID 原地更新，让用户看到系统正在做什么。

### 7.3 工具调用与待确认机制

系统内置日程、计划、心理、饮食、本地生活、搜索等工具。对于会改变用户数据的动作，系统通过 pending action 机制让用户确认后再写入。

### 7.4 旁路学习与后台任务

主对话返回后，记忆提取和偏好学习可以在后台继续进行，不阻塞用户响应。Shadow Learner 以旁路方式沉淀偏好，形成长期画像。

### 7.5 RAG 与知识能力

项目包含 RAG 管线、文件解析、切分、向量检索和知识库搜索能力。当前前端知识库入口暂时替换为个人画像页，RAG 能力保留为后续恢复和扩展方向。

### 7.6 Java / Python / Web 多服务架构

项目包含：

- `shadowlink-web`：React 前端；
- `shadowlink-ai`：Python FastAPI AI 服务；
- `shadowlink-server`：Java Spring Boot 网关与业务层；
- `shadowlink-electron`：桌面壳方向。

这使项目既能演示 AI 能力，也具备向桌面产品和完整后端服务演进的基础。

## 8. 当前可演示能力

当前推荐重点演示：

1. 私聊真实执行步骤：展示系统不是黑箱回复；
2. 日程与长期计划：从一句话目标到计划日和日历；
3. 心理关怀：压力表达、轻量陪伴、趋势和反馈；
4. 圆桌决策：复杂问题多视角讨论和结构化结果；
5. 本地生活机会：结合状态、天气、位置和空档做建议；
6. 个人画像：长期偏好、记忆和 Shadow Learner。

## 9. 比赛表达建议

建议开场不要说“我们做了一个 AI 助手”。可以这样说：

> 我们做的不是一个更会聊天的助手，而是一个个人生活操作系统。它把日程、心理、长期计划、现实环境和个人记忆连接在一起，让 AI 不只回答问题，而是参与个人生活中的判断、安排、执行和后续反馈。

接着用一句对比强化差异：

> 普通助手回答“你可以怎么做”，Jarvis-Life 进一步回答“你现在适不适合做、应该什么时候做、谁来判断、如何写入系统、之后如何继续跟进”。

最后用 demo 证明闭环，而不是逐个罗列功能。
