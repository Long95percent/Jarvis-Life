## 二、技术人员下一步应该做什么

### 第一优先级：心理机制从 MVP 走向全量

对应 checklist：`docs/lsq-worklog/4-28-worklog/原始设计完成度-checklist.md` 的“五、05 心理关怀模块 Checklist”。

建议按以下顺序实现：

1. `B1 情绪采集层`
   - 新增 `jarvis_emotion_observations`。
   - 把当前 `mood_care.py` 的规则结果写入 observation。
   - 增加必要时 LLM 结构化 JSON。
   - 不保存心理原文全文，只保存摘要和指标。

2. `B4 每日心理快照层`
   - 新增 `jarvis_mood_snapshots`。
   - 聚合当天 emotion observations、LifeContext、任务压力初步数据。
   - 注意区分“单轮状态卡”和“日级 mood snapshot”。

3. `B2 行为采集层`
   - 新增 `jarvis_behavior_observations`。
   - 记录首次活跃、最后活跃、深夜使用、超过 bedtime。

4. `B3 日程压力观测层`
   - 新增 `jarvis_stress_signals`。
   - 读取 calendar、background_task_days、workbench、missed tasks。
   - 输出可解释 reason。

5. `B5-B7 趋势、触发、交互`
   - `/care/trends`。
   - cooldown / daily budget。
   - 用户反馈闭环。

### 第二优先级：圆桌 decision / brainstorm 产品化

对应 checklist：`七、07 圆桌模式 Checklist`。

做完心理基础后，圆桌才能真正消费心理状态。下一步请实现：

1. `mode=decision|brainstorm`。
2. `decision_result` 结构化输出。
3. `brainstorm_result` 结构化输出。
4. accept / return / save / handoff 接口。
5. decision 接受后交给 Maxwell 生成 pending action。

### 第三优先级：轻量管家团队和计划闭环

对应 checklist：`六、06 轻量管家团队 Checklist` 和 `四、04 整体架构闭环 Checklist`。

重点不是做更多 Agent，而是明确边界：

- 什么场景直接回复。
- 什么场景 consult。
- 什么场景推荐圆桌。
- 什么场景交给 Maxwell。
- 所有写操作必须 pending confirmation。

## 三、防缩水硬规则

技术实现时必须遵守：

1. 任何 worklog 不能写“完成心理模块”，除非 checklist 中心理模块全部 `[DONE]`。
2. 如果只完成一层，标题必须写“心理机制 B1 情绪采集层 MVP”之类的具体范围。
3. 每次提交必须说明 checklist 哪些项从 `[TODO]` 变成 `[MVP]` 或 `[DONE]`。
4. 不允许只做前端卡片就宣称完成后端闭环。
5. 不允许只做规则关键词就宣称完成情绪分析。
6. 不允许只做单轮 action 就宣称完成每日 mood snapshot。
7. 不允许只做 roundtable start/continue 就宣称完成圆桌模式。
8. 不允许只做 consult prompt 就宣称完成轻量管家团队。
9. 所有心理相关高风险场景必须保留安全边界：不诊断、不替代医生，必要时提示联系可信任的人和当地紧急渠道。
10. 每做完一个模块必须补测试或验证文档。

## 五、推荐给技术人员的第一条具体任务

请实现 `心理机制 B1：情绪采集层 MVP`。

范围：

- 新增 `jarvis_emotion_observations` 持久化。
- 当前 `mood_care.detect_mood_snapshot()` 的结果必须写入 observation。
- observation 不保存用户原文全文，只保存 `evidence_summary` 和结构化指标。
- 给 Mira 私聊链路补测试：低能量、压力、睡眠、高风险、无心理信号。
- worklog 明确写：这是 B1 MVP，不是心理机制全量完成。

验收：

- 用户向 Mira 说“我今天特别累，不想学了，也有点自责”，后端写入一条 emotion observation。
- 该 observation 包含 primary_emotion / stress_score / fatigue_score / risk_level / evidence_summary。
- `mood.snapshot` 和 `care.intervention` 仍正常返回前端。
- 高风险关键词触发 high risk，但不输出医疗诊断。
- 测试或验证文档能复现。
