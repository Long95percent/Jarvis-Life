## 二、心理关怀前端

### 已完成

- 在私聊消息 actions 下支持渲染心理类 action。
- 支持 `mood.snapshot`、`care.intervention`、`care.followup` 三类卡片。
- 新增独立 `CareCard` 组件，承载心理状态、关怀建议、回访、反馈操作。
- 关怀卡片支持用户操作：稍后提醒、我已处理、有帮助、太频繁、不需要此类提醒等。
- `jarvisApi` 增加 care feedback 相关调用，用于把用户操作回写后端。
- `ProactiveMessageFeed` 配合关怀消息状态展示，支持 read/dismiss/snooze 类状态。

### 用户能看到什么

- 用户和 Mira 聊“我今天特别累，不想学了”后，不只是看到一段文字，还能看到心理状态记录和 Mira 关怀建议卡片。
- 用户要求“晚点提醒我休息”后，可以看到后续回访卡片，并能点击反馈按钮。
- 高风险表达会展示更谨慎的安全提示，不做医疗诊断。

### 仍需注意

- 需要确认 `CareCard` 与父组件不会重复发送 feedback 请求。
- “稍后提醒”需要确认后端 snooze 状态能正确影响 proactive feed。
- 后续最好把心理卡片状态和 proactive message 状态进一步统一。

## 三、心理趋势 / 心理中心前端

### 已完成

- 新增 `CareTrendsPanel`，用于展示心理趋势数据。
- `jarvisApi` 增加 `/care/trends`、心理设置、行为观测等相关 API 调用。
- 页面保留正常心理趋势中心入口，不再把它当 Demo 面板。
- 支持从用户视角查看 mood、stress、energy、sleep risk、schedule pressure 等趋势信息。

### 用户能看到什么

- 用户可以进入心理趋势/心理中心，看最近一周或一段时间的状态变化。
- 用户可以看到心理状态不是单次回复，而是长期趋势的一部分。

### 仍需注意

- 需要确认心理中心入口足够明显，普通用户不用看 DevTools 也能找到。
- 需要确认点击某一天能看到可解释详情，例如低能量表达、晚睡、任务压力等。
- 关闭心理追踪后，前端不能展示伪数据。

## 四、圆桌会议前端

### 已完成

- `RoundtableStage` 支持 Decision / Brainstorm 两种结果展示。
- 圆桌保留角色轮流发言形式，不改成一次性输出整轮。
- 强化席位、头像、当前发言高亮和 `Speaking` 标识。
- 中心圆桌增加主持总结卡。
- 当前发言气泡做长度控制，完整内容保留在“完整记录”中，避免舞台文本墙。
- Decision 结果卡展示推荐结论、利弊、下一步动作。
- Brainstorm 结果卡展示主题、想法、张力、追问。
- Brainstorm 支持保存灵感、继续讨论、转成计划。
- Decision 支持接受建议 / 让 Maxwell 执行，生成待确认动作。
- 前端显示 roundtable timing 总耗时，用于解释轮流发言耗时。

### 用户能看到什么

- 用户进入圆桌后，可以看到角色轮流发言，而不是一整段冷冰冰总结。
- Decision 圆桌能给出推荐方案和后续动作。
- Brainstorm 圆桌能发散想法，并允许保存灵感或转成计划。
- 圆桌页面有更明确的模式感和舞台感。

### 仍需注意

- `返回私聊` 链路还需要后端接口完整支持，前端按钮要真正把总结带回来源私聊。
- 需要补单个 Agent 发言失败时的前端失败/跳过说明。
- 需要补更明确的进度反馈，例如第几位 / 共几位 / 正在生成总结。

## 五、统一计划 / 日历前端

### 已完成

- `jarvisApi` 增加 planner 相关接口调用。
- `CalendarPanel` 接入 planner calendar items，用于合并展示计划 day 和日历项。
- 前端开始支持统一计划领域，不再只依赖旧的 background task day 展示。
- 支持查看计划 day、日历投影、Maxwell 工作台相关信息。

### 用户能看到什么

- 长期计划拆成每日任务后，可以在日历/任务相关视图里被看到。
- Maxwell 待确认或工作台任务可以和计划 day 产生关联。

### 仍需注意

- 需要页面验收：普通日历事件、计划 day、background task day 是否合并显示正确。
- 需要确认完成、延期、重排后的状态能同步刷新。
- 需要确认心理压力和圆桌 decision 能读取计划负载并在页面解释。
