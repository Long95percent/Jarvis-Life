# StepE Planner Maintenance Closure (ASCII copy)

Date: 2026-04-28
Scope: finish the remaining gap: after overdue days become missed, Maxwell/LLM generates a new future plan and the daily scheduler invokes it.

## Completed

- Added `shadowlink-ai/app/jarvis/planner_maintenance.py`.
- Daily maintenance chain now runs: mark overdue missed -> group missed plan days by plan -> call Maxwell/LLM -> update future unfinished plan days -> sync existing calendar events -> push today's planner days to Maxwell workbench.
- LLM is constrained to modify future unfinished `jarvis_plan_days` only; `missed`, `completed`, and historical days are preserved.
- If LLM is unavailable or returns invalid JSON, deterministic fallback shifts future days by missed count, so the chain stays usable.
- Automatic reschedule writes `plan.rescheduled` events and creates a Maxwell proactive message for the user.
- `ProactiveRoutineScheduler.check_routines()` calls `run_planner_daily_maintenance_once()` once per local date after 01:00.
- Manual APIs were added for operation and roadshow: `/jarvis/planner/daily-maintenance` and `/jarvis/planner/daily-maintenance/once`.
