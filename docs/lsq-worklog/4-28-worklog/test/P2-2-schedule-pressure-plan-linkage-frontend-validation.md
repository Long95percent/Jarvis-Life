# P2-2 日程压力完整来源与计划联动前端验证方法

## 验证目标

验证心理趋势与圆桌决策能真实体现计划负载，而不是只看聊天情绪。用户应能在前端看到：任务密度、未完成/逾期、计划重排、连续高压天数如何影响当天压力解释。

## 路演 Demo 准备

1. 打开 Jarvis 正常使用页面，不打开后台调试面板。
2. 确认心理追踪开启。
3. 准备一个 Maxwell 长计划，包含多天任务，其中至少一天任务较多或预计时长较长。
4. 准备一个 Decision 圆桌问题，例如：“我今晚很累，但还有学习计划，要不要继续？”

## Demo 操作脚本

### 场景 1：当天计划负载进入心理趋势

1. 在 Maxwell 创建或确认一个当天任务较多的计划。
2. 保持当天有多个未完成计划项或工作台 todo/doing 项。
3. 回到心理趋势中心，点击当天。

预期表现：

- 当天详情解释中出现任务/日程压力。
- 压力来源能指向计划项、工作台事项或日程事件。
- snapshot 的 `schedule_pressure_score` 会反映计划负载。

### 场景 2：missed 后自动重排形成压力来源

1. 制造一个昨天或前一天未完成的计划项，让维护任务把它标记为 missed。
2. 等待 Maxwell 自动重排后续计划，或重启/等待 routine 触发维护。
3. 打开心理趋势中心点击重排当天。

预期表现：

- 当天详情能出现计划重排/重排失败相关压力说明。
- 解释中能看到这是来自计划事件日志，而不是用户聊天原文。
- 关怀卡片如果出现，应说明“你没有完成计划，已经重排”，不要求用户确认重排。

### 场景 3：连续高压天数

1. 连续几天安排较密任务或高压力日程。
2. 每天正常使用 Mira 或让 mood snapshot 自动生成。
3. 第三天打开心理趋势中心，点击当天。

预期表现：

- 当天压力解释中出现连续高压提示。
- 这个提示基于最近多天 snapshot 的 stress/schedule pressure，不是前端临时猜测。

### 场景 4：Decision 圆桌体现计划负载

1. 在 Mira 或 Alfred 私聊中发起：“我今晚很累，但还有学习计划，要不要继续？”
2. 进入 Decision 圆桌。
3. 查看圆桌上下文解释或专家发言。

预期表现：

- 圆桌能参考今日心理状态、任务压力、关键日程/计划负载。
- 专家建议不只是情绪安慰，也能围绕计划减负、顺延、保留关键任务等给出方案。

## 验收口径

- 必须通过前端正常页面验证：Maxwell 计划/工作台、心理趋势中心、Decision 圆桌。
- 不把后台接口返回当作唯一验收。
- 本步骤只代表 P2-2 完成；P2-3 心理中心产品化入口仍需继续。
