# P1-2 mood snapshot 自动生成与 backfill 前端验证方法

## 验证目标

验证心理日快照不再依赖手动调用后台接口，而是在正常使用 Jarvis/Mira、完成任务、重启或等待例行检查后自动生成，并能在心理趋势/关怀入口里被用户理解。

## 路演 Demo 准备

1. 打开前端正常使用页，不打开任何调试面板或后台接口页面。
2. 确认心理追踪开关处于开启状态。
3. 准备一个当天计划或 Maxwell 工作台事项，例如“完成路演讲稿”。
4. 准备一段和 Mira 的自然对话，不要说明是在测试后台逻辑。

## Demo 操作脚本

### 场景 1：当天快照自动形成

1. 在 Mira 私聊输入：“今天上午压力有点大，但我刚刚把路演讲稿完成了，现在轻松很多。”
2. 在任务/计划页把“完成路演讲稿”标记为已完成。
3. 继续在 Mira 私聊输入：“我今晚会按时休息，明天再处理剩下的内容。”
4. 重启应用，或保持应用运行到下一次后台例行检查。
5. 打开心理趋势/关怀中心页面，查看今天是否出现 mood snapshot 或趋势记录。

预期表现：

- 不需要用户点击“生成快照”或访问 `/care/snapshots`。
- 今天的心理趋势出现压力、能量、主要情绪、风险标记等摘要。
- 正向事件里能看到类似“完成任务”“表达积极情绪”“按时休息”的来源。

### 场景 2：缺失日期 backfill

1. 连续两天正常使用 Mira，分别表达一次压力或疲劳状态。
2. 第二天重启应用，或让应用保持到后台例行检查触发。
3. 打开心理趋势页面，切换最近几天视图。

预期表现：

- 最近 N 天中有有效心理/行为/压力数据但缺快照的日期会被补齐。
- 已经存在的历史快照不会被无意义重复刷新；当天因为信息仍在累计，可以更新。

### 场景 3：正负事件一起解释当天状态

1. 上午对 Mira 说：“今天有点焦虑，会议太密了。”
2. 下午完成一个计划事项，并对 Mira 说：“做完以后放松很多。”
3. 打开心理趋势/关怀中心查看当天记录。

预期表现：

- snapshot 不只显示负面压力，也能展示完成任务、放松表达等正向事件。
- 关怀卡片或趋势说明不会变成后台日志展示，而是面向用户解释“为什么今天压力/能量是这个状态”。

## 验收口径

- 只通过前端正常页面完成验证：Mira 私聊、任务/计划完成、心理趋势/关怀中心。
- 后端单测只是辅助证据，不能替代前端路演验证。
- 本步骤仍只是 P1-2 完成，不代表心理模块全量完成；P1-3 详情解释链路、P2 行为/压力/心理中心仍需继续。
