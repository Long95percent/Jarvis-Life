# P1-3 心理趋势某日详情解释前端验证方法

## 验证目标

验证用户在心理趋势中心点击某一天时，能看到“为什么这天压力高/能量低”的可读解释，而不是后台字段堆叠或调试面板。

## 路演 Demo 准备

1. 打开 Jarvis 前端正常使用页，进入 Mira 私聊、任务/计划页、心理趋势中心。
2. 确认心理趋势追踪处于开启状态。
3. 准备一个当天较密集的计划或任务，例如“路演材料、会议准备、复盘材料”。

## Demo 操作脚本

### 场景 1：解释压力来源

1. 在 Mira 私聊输入：“今天会议和路演准备都挤在一起，我有点焦虑，感觉压力很高。”
2. 在任务/计划页保留多个当天任务，或者让当天有未完成/高密度任务。
3. 等待自动快照生成，或重启应用触发启动检查。
4. 打开心理趋势中心，选择“压力”或“日程压力”指标。
5. 点击当天柱状点。

预期表现：

- 展开区显示当天来源解释。
- 解释里能看到任务/日程压力、聊天中的高压表达、snapshot summary。
- 不是展示后台 JSON，而是自然语言解释“为什么这天压力高”。

### 场景 2：解释能量低和作息因素

1. 晚上或模拟晚间使用时，对 Mira 说：“今天太累了，感觉没什么精力。”
2. 如果前端支持作息/行为记录，产生超过 bedtime 的活跃记录。
3. 回到心理趋势中心，点击当天。

预期表现：

- 展开区显示低能量表达。
- 如果有晚睡/超过 bedtime 行为，解释会出现作息信号。
- 风险标签只作为辅助标签，不替代自然语言说明。

### 场景 3：关怀触发和正负事件一起出现

1. 先制造一个高压日：任务密集、压力表达、晚睡或低能量表达。
2. 完成其中一个任务，让当天同时存在正向事件。
3. 等待关怀卡片自动出现，或触发后台 routine。
4. 在心理趋势中心点击当天。

预期表现：

- 展开区能同时看到正向事件、负向来源、关怀触发类型。
- 用户能理解“系统为什么提醒我休息/减负”。
- 关闭心理追踪后，趋势中心不展示历史伪详情，只提示追踪已关闭。

## 验收口径

- 必须通过前端正常页面验证：Mira 私聊、任务/计划页、心理趋势中心。
- 用户点击某天能看到压力来源、作息/低能量/关怀触发等解释。
- 后端单测和接口只是辅助证据，不能替代前端路演验证。
- 本步骤只代表 P1-3 完成；心理模块 P2 行为可靠性、压力全来源、心理中心产品化仍需继续。
