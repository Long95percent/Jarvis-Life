# 圆桌返回私聊与降级进度前端验证手册

## 目标
验证圆桌不是后台展示面板，而是用户可完整操作的真实链路：从私聊触发圆桌、角色轮流发言、看到进度/降级提示、把总结带回原私聊继续。

## 准备
1. 启动后端与前端。
2. 确认 Jarvis 页面右侧不是调试面板，而是正常使用入口。
3. 选择一个可触发圆桌的私聊场景，例如 Mira 或 Alfred 私聊。

## 路演 Demo 脚本
### Demo 1：从私聊发起 Decision 圆桌并回到私聊
1. 打开 Mira 私聊。
2. 输入：`我今天很累，但晚上还想学习，帮我判断要不要继续。`
3. 若页面出现圆桌建议入口，进入圆桌；或从场景入口选择“疲惫学习决策”。
4. 观察圆桌舞台：角色必须轮流发言，不应一次性出现整轮完整内容。
5. 等待右侧 Decision 结果卡出现。
6. 点击 `带总结回私聊`。
7. 预期：页面返回原私聊 session，并出现一条“圆桌讨论总结”消息。
8. 在私聊继续追问：`那我现在第一步做什么？`
9. 预期：原 Agent 能基于圆桌总结继续回答。

### Demo 2：Brainstorm 圆桌保存/转计划/回私聊
1. 从场景入口选择“工作难题头脑风暴”。
2. 输入：`我想做一个能展示 Jarvis 价值的路演 demo。`
3. 等待角色轮流发言与 Brainstorm 结果卡。
4. 可分别演示：
   - 点击 `保存灵感`：结果卡显示已保存记忆。
   - 点击 `转成计划`：生成 Maxwell 待确认计划卡，不直接改计划。
   - 点击 `带总结回私聊`：回到来源私聊继续。

### Demo 3：进度与降级体验
1. 正常圆桌进行时，观察顶部状态栏。
2. 预期能看到类似 `进度 1/4 · 发言中`、`进度 4/4 · 已完成`。
3. 如果某个模型/Provider 临时失败，页面左上会出现“某角色已降级继续”。
4. 预期：失败不会卡死整轮，后续角色继续发言。

## 通过标准
- 圆桌始终是角色轮流发言，不是一次请求返回整轮。
- `带总结回私聊` 后，原私聊出现圆桌总结消息。
- Decision/Brainstorm 结果卡会展示交接状态，避免重复操作误解。
- 单 Agent 失败时，用户能看到降级提示，且圆桌继续。

## 当前剩余限制
- 若圆桌不是从私聊发起，后端可能没有 `source_session_id`，此时 `带总结回私聊` 会提示无法返回来源私聊；这属于正确保护。
- 降级提示依赖后端捕获单 Agent 失败，正常环境不一定每次都能自然触发。

## 补充验证：来源字段与解释链路
1. 从 Mira 私聊里触发圆桌，而不是直接从场景广场打开。
2. 等 Decision 结果卡出现后，检查卡片里是否有“为什么这样建议”。
3. 预期能看到心理状态、日程压力、今日计划三类解释。
4. 点击 `带总结回私聊`。
5. 预期回到刚才的 Mira 私聊，并看到圆桌总结消息，而不是打开一个新私聊。
