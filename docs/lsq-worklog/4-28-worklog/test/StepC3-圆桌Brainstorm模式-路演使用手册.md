# StepC3 圆桌 Brainstorm 模式路演使用手册

## 路演故事

用户想设计一个能打动观众的 Jarvis 展示方案，但还没有确定方向。Jarvis 召集 Brainstorm 圆桌做发散：先产生想法，再由用户决定是保存为灵感，还是交给 Maxwell 变成待确认计划。

## 路演步骤

### 1. 提出发散问题

进入圆桌场景，选择“工作难题头脑风暴”，输入：

```text
我想做一个最能展示 Jarvis 潜力的路演 demo，应该怎么设计？
```

对观众说明：

> “这不是让系统马上排计划，而是先让不同角色帮我打开思路。”

### 2. 展示轮流发散

观察舞台上的角色：

- Moderator 负责框定问题。
- Explorer 负责提出更大胆的创意。
- Critic 负责指出风险和可行性问题。
- Synthesizer 负责把想法收束成方向。

讲解重点：

> “Brainstorm 模式只发散，不默认执行。它不会偷偷写计划或改日程。”

### 3. 展示 Brainstorm 结果卡

等待右侧 Brainstorm 卡出现，展示：

- 主题。
- 候选想法。
- 冲突 / 张力。
- “不会自动写计划或改日程”的提示。

### 4. 保存灵感

点击“保存灵感”。

预期用户可见结果：

- 卡片提示已保存为记忆。
- 后续 Jarvis 可以在对话中回忆这次灵感。
- 不会生成计划，不会改日程。

### 5. 转成计划

如果观众想看下一步执行链路，点击“转成计划”。

预期用户可见结果：

- 页面提示已生成 Maxwell 待确认计划卡。
- 用户仍需在待确认动作中最终确认。
- 确认前不会写入正式计划或日程。

## 成功标准

路演时观众应该能理解：

- Brainstorm 是发散模式，不是执行模式。
- 用户可以把结果保存为长期灵感。
- 用户也可以主动把结果交给 Maxwell 变成计划。
- 所有写入动作都有用户确认边界。

## 推荐串联

可以先演示 C3 Brainstorm：生成路演 demo 创意；再演示 C2 Decision：当用户疲惫但仍有任务时，Jarvis 如何做多角色决策。这样能体现 Jarvis 同时支持“发散”和“决策”。
