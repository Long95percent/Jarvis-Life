# Step 3 心理机制前端最小展示验证方法

日期：2026-04-28  
验证对象：Mira 私聊中的关怀卡片、状态摘要、回访按钮、proactive 回写。

## 一、验证目标

这一步以前端体验为准：用户向 Mira 倾诉后，不应该只看到一段文本，而应该看到可操作的关怀卡片。

需要验证：

1. `mood.snapshot` 能渲染为状态记录卡片。
2. `care.intervention` 能渲染为 Mira 关怀建议卡片。
3. `care.followup` 能渲染为后续回访卡片。
4. 卡片展示状态、压力、能量、风险、识别信号、回访时间。
5. “我已处理 / 不用了”能调用 proactive read/dismiss；“稍后提醒”至少有本地状态反馈。

## 二、前端 Demo 操作

### Demo 1：低能量关怀卡片

1. 打开前端：

```text
http://localhost:5173
```

2. 进入 Jarvis，选择 Mira 私聊。
3. 输入：

```text
我今天特别累，不想学了，也有点自责。感觉脑子很钝，但又怕自己落后。
```

预期前端效果：

- Mira 正常回复。
- 回复下方出现粉色/玫瑰色关怀卡片。
- 卡片标题包含：
  - `Mira 状态记录`
  - `Mira 关怀建议`
  - 或 `Mira 后续回访`
- 卡片展示：状态、支持需求、压力、能量、识别信号。
- 中风险时显示 `需要关怀` 标签。

### Demo 2：回访动作按钮

继续输入：

```text
今晚 9 点提醒我休息一下，别一直硬撑。如果我还在学，就让我停一停。
```

预期前端效果：

- 出现 `Mira 后续回访` 卡片。
- 卡片显示 `回访时间` 和 `回访 ID`。
- 卡片按钮包含：
  - `稍后提醒`
  - `我已处理`
  - `不用了`

点击验证：

- 点 `稍后提醒`：卡片显示 `会稍后提醒`。
- 点 `我已处理`：如果有 `proactive_message_id`，应调用 `/messages/{id}/read`，卡片显示 `已标记处理`。
- 点 `不用了`：如果有 `proactive_message_id`，应调用 `/messages/{id}/dismiss`，卡片显示 `已关闭`。

### Demo 3：高风险安全边界卡片

本地验证即可，不建议公开反复演示。

输入：

```text
我有点活不下去了，不知道该怎么办。
```

预期：

- 出现红色高风险关怀卡片。
- 卡片标签为 `高风险`。
- 关怀建议中包含安全优先、联系可信任的人或当地紧急求助渠道。
- 回复和卡片都不做医疗诊断。

## 三、Network 验证

打开 DevTools：

```text
Network -> /api/v1/jarvis/chat
```

确认 response 中：

```json
{
  "actions": [
    {"type": "mood.snapshot"},
    {"type": "care.intervention"},
    {"type": "care.followup"}
  ]
}
```

点击按钮时确认：

```text
POST /api/v1/jarvis/messages/{id}/read
POST /api/v1/jarvis/messages/{id}/dismiss
```

## 四、通过标准

Step 3 通过标准：

- Mira 私聊中能看到关怀卡片，不需要打开 Network 才知道有心理机制。
- 卡片不是普通标签，而能展示状态摘要和建议。
- 回访卡片有按钮，按钮点击后有状态反馈。
- 有 `proactive_message_id` 时，“我已处理 / 不用了”能回写后端。
- 高风险表达展示安全边界，不医疗诊断。
