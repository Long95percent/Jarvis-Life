# P2-3 心理中心产品化入口前端验证方法

## 验证目标

验证用户能在 Jarvis 正常主页直接使用心理中心：查看今日状态、趋势图、最近关怀、某日解释、隐私开关和清除数据。页面不出现 Demo/MVP/后台调试字样。

## 路演 Demo 准备

1. 打开 Jarvis 主页右侧栏。
2. 确认右侧出现“心理中心”卡片。
3. 准备当天至少一条 Mira 情绪对话、一个计划/任务完成或未完成事件，以便产生今日状态。

## Demo 操作脚本

### 场景 1：从主页进入心理中心

1. 打开 Jarvis 主页。
2. 观察右侧栏最上方心理中心卡片。
3. 查看心情、压力、能量三个今日状态数字。
4. 点击“找 Mira 聊聊”。

预期表现：

- 用户不需要打开调试面板或接口页面。
- 心理中心是正常产品入口，文案面向用户。
- “找 Mira 聊聊”能进入 Mira 私聊。

### 场景 2：查看趋势和某日解释

1. 在心理中心选择“本周/本月/全年”。
2. 切换心情、压力、能量、睡眠、计划压力指标。
3. 点击某一天的柱状点或热力点。

预期表现：

- 趋势图跟随指标切换。
- 点击某天后出现来源解释，包括压力信号、行为信号、情绪证据、关怀触发、正负事件。
- 解释是自然语言，不展示后台 JSON。

### 场景 3：最近关怀

1. 制造一个高压/晚睡/计划重排场景，让 Mira 产生关怀卡片。
2. 回到 Jarvis 主页右侧心理中心。

预期表现：

- 心理中心显示最近关怀摘要。
- 主动提醒列表仍保留完整 CareCard 操作，不重复制造新的展示面板。

### 场景 4：隐私开关与清除数据

1. 点击“心理追踪已开启”，切换为关闭。
2. 继续正常聊天或切换页面。
3. 回到心理中心查看提示。
4. 点击“清除心理数据”，确认后观察趋势清空。

预期表现：

- 关闭追踪后不会继续写入新的情绪、行为和压力信号。
- 清除数据后趋势和详情不再显示旧心理数据。
- 页面没有 Demo/MVP/debug 字样。

## 验收口径

- 验证必须通过 Jarvis 主页、Mira 私聊、心理中心正常产品流程完成。
- 心理中心需要同时覆盖今日状态、趋势图、最近关怀、详情解释、隐私开关、清除数据。
- 本步骤完成后，当前指令中心理模块/圆桌模块范围内的 P0/P1/P2-1/P2-2/P2-3 均已逐项完成；若后续要做速度调优或管家团队架构升级，需要另起计划。
