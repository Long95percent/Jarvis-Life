# 生活状态与心理中心指标说明

日期：2026-05-03

## 1. 生活状态卡片里的 3 个指标

位置：`shadowlink-web/src/components/jarvis/DashboardCards.tsx`

- **压力**：直接读取后端 `LifeContext.stress_level`，范围 0-10。
- **日程**：直接读取后端 `LifeContext.schedule_density`，范围 0-10。
- **睡眠**：直接读取后端 `LifeContext.sleep_quality`，范围 0-10。
- **心情图标**：直接读取 `LifeContext.mood_trend`，映射成正向 / 中性 / 负向表情。

结论：生活状态卡片本身**不做复杂计算**，主要是把后端生活上下文直接展示出来。

## 2. 这些值从哪来

位置：`shadowlink-ai/app/jarvis/models.py`、`shadowlink-ai/app/jarvis/context_bus.py`

- `LifeContext` 是统一上下文模型，字段包含：
  - `stress_level`
  - `schedule_density`
  - `sleep_quality`
  - `mood_trend`
- `LifeContextBus.update_fields()` 会更新上下文字段，并把当前状态持久化。

## 3. 心理中心里的指标怎么计算

位置：`shadowlink-ai/app/jarvis/mood_snapshot.py`

心理中心展示的主要是每日快照，核心字段有：

- **心情**：`mood_score`
  - 由当天情绪 observation 的 `valence` 平均值换算得到，范围 0-10。
- **压力**：`stress_score`
  - 由当天情绪 observation 里的 `stress_score` 平均得到。
- **能量**：`energy_score`
  - 先算 `fatigue_score`，再用 `10 - fatigue_score` 得出能量。
- **睡眠**：`sleep_risk_score`
  - 结合晚间使用、睡眠信号等行为/情绪证据，越高表示睡眠风险越高。
- **计划压力**：`schedule_pressure_score`
  - 来自日程压力信号里的最高分。

## 4. 日程压力是怎么来的

位置：`shadowlink-ai/app/jarvis/stress_observation.py`

这里会综合：

- 日历事件的持续时长 × `stress_weight`
- 当天的计划/后台任务/待确认动作
- 连续高压力天数
- 行为信号（例如晚间活跃、睡眠边界等）

最终会生成压力信号，再汇总成 `schedule_pressure_score`，并写入每日快照。

## 5. 心理中心页面展示来源

位置：`shadowlink-web/src/components/jarvis/CareTrendsPanel.tsx`

页面展示的内容主要来自：

- `mood_score`
- `stress_score`
- `energy_score`
- `sleep_risk_score`
- `schedule_pressure_score`

这些值不是前端算的，而是后端每日快照直接返回的。

## 6. 一句话总结

- **生活状态卡片**：直接展示当前生活上下文。
- **心理中心**：展示每日聚合后的情绪、压力、能量、睡眠和计划压力趋势。
- **大部分指标都在后端算好**，前端负责显示。
