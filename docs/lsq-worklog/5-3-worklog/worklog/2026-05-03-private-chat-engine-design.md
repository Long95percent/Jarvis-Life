# 2026-05-03 私聊智能体引擎化设计日志

## 背景

当前 Jarvis 私聊已经有角色分工和工具白名单，但私聊执行链路主要是轻量工具循环：模型输出工具块，后端执行工具，再让模型总结结果。

这个链路能处理简单工具调用，但遇到复杂写操作时不够稳。例如日程更新缺少 `event_id` 时，模型会承认参数漏传，但系统不会自动进入“先查询目标、补齐 ID、再执行更新”的 ReAct 式修复流程。

项目中已有通用 AgentEngine，包含 `DIRECT`、`REACT`、`PLAN_EXECUTE` 等策略能力，但它还没有完整接到 Jarvis 私聊链路中。

## 用户目标

- 只改私聊工具链，不重做前端聊天入口。
- 所有私聊智能体都获得引擎能力，而不是只增强 Maxwell。
- 每个角色的边界必须保持解耦：人格、职责、工具权限不能混在一起。
- 引擎只是赋能执行方式，不改变角色分工。
- 后续涉及接口或返回结构变化时，必须同步更新 `docs/解耦接口说明`。

## 设计原则

### 1. 角色边界不动

每个智能体继续由 `shadowlink-ai/app/jarvis/agents.py` 定义：

- `system_prompt` 决定角色人格和说话风格。
- `tool_whitelist` 决定可用工具边界。
- `proactive_triggers` 和 `interrupt_budget` 决定主动触发边界。

通用引擎不得绕过这些边界。即使进入 `REACT` 或 `PLAN_EXECUTE`，也只能使用当前角色白名单内的工具。

### 2. 引擎能力作为执行层

私聊链路中新增一个“Jarvis 私聊引擎适配层”。它只负责决定如何执行：

- 简单问答：走 `DIRECT`。
- 需要查工具、补参数、重试的任务：走 `REACT`。
- 多步骤计划、长期任务、复杂日程重排：走 `PLAN_EXECUTE`。

适配层不拥有业务逻辑，业务仍落在日程、心理、营养、活动等后端模块和工具中。

### 3. 保持前后端解耦

前端仍然只调用私聊接口和 service 层，不参与工具补参、冲突判断、数据库写入。

如果智能体多跑几轮工具，前端只展示后端返回的真实步骤、耗时、工具名、结果摘要。

### 4. 写操作必须可追踪

所有写操作要保留真实工具结果，不能只让模型口头承诺。

返回给前端的 action/step 至少能说明：

- 选择了哪个策略。
- 调用了哪些工具。
- 是否自动补参。
- 是否真正写入成功。
- 如果失败，失败原因是什么。

## 推荐方案

采用“角色配置 + 引擎执行”的混合模式。

### 流程

1. 前端仍调用现有私聊接口。
2. 后端根据 agent_id 读取角色配置。
3. 后端构造 JarvisEngineRequest，包含：用户消息、角色 prompt、工具白名单、上下文、历史、当前模式。
4. 策略选择器判断使用 `DIRECT` / `REACT` / `PLAN_EXECUTE`。
5. 引擎执行时只能调用该角色白名单工具。
6. 工具失败时，引擎可根据失败原因追加查询、补参、重试。
7. 后端把最终回复、工具结果和真实步骤返回给前端。

## 每个角色的默认倾向

| 角色 | 默认倾向 | 升级条件 |
| --- | --- | --- |
| Alfred | `DIRECT` | 跨模块协调、用户要求执行动作、需要查多个模块时升级 `REACT` |
| Maxwell | `REACT` | 长期计划、批量日程修改、延期重排升级 `PLAN_EXECUTE` |
| Nora | `DIRECT` | 需要结合日程、天气、饮食限制、外部资料时升级 `REACT` |
| Mira | `DIRECT` | 需要读取压力、睡眠、日程并安排回访时升级 `REACT` |
| Leo | `DIRECT` | 需要查活动、路线、天气、空闲时间时升级 `REACT` |
| Athena | `DIRECT` | 学习计划、复盘、长期任务分解升级 `PLAN_EXECUTE` |

## 风险控制

- 第一阶段只改私聊链路，不改圆桌。
- 不让引擎访问角色白名单之外的工具。
- 不让前端参与补参和数据库写入。
- 每次工具调用必须落到真实 tool result，不能只在模型回复里说“已完成”。
- 先用 Maxwell 日程删改作为验收样例，再推广到其他角色。

## 后续实施建议

1. 新增 Jarvis 私聊引擎适配层，不直接改通用 AgentEngine 核心。
2. 先让适配层复用现有 `execute_tool_calls`，保证白名单和 ToolRegistry 不变。
3. 增加策略选择字段和真实步骤返回字段。
4. 为缺参工具调用增加 ReAct 修复：缺 `event_id` 时先 query，再 update/delete。
5. 更新接口说明，告诉前端新增的步骤字段如何展示。
6. 加回归测试：所有角色不可越权，Maxwell 可自动补参更新日程。
