# 2026-04-26 确认日程卡与历史对话二次修复

## 问题

用户真实前端测试发现：

1. Maxwell 的确认日程卡看不到。
2. 和其它智能体聊天时让秘书安排，日程卡也看不到。
3. 同时弹出 Brainstorm / 团队协作提示，疑似覆盖或干扰日程卡。
4. 私聊点击返回后，历史对话区域没有立即展示记录。

## 修复

- 前端 `sendMessage` 识别后端返回的 `response.agent_id`：如果请求从其它 Agent 路由到 Maxwell，会把 Maxwell 回复和 actions 同步进当前聊天与 Maxwell 聊天，并切换 activeAgent 到 Maxwell，确保确认卡展示在当前产品界面。
- 后端新增跨 Agent 日程兜底：如果已识别 schedule/task intent 但模型没有吐出工具调用，后端会兜底生成 `jarvis_calendar_add` 或 `jarvis_task_plan_decompose` 的待确认 action，避免只返回文字没有卡片。
- 前端发送后如果最新回复含 `pending_confirmation`，不再展示 Brainstorm / 团队协作提示，避免干扰确认卡。
- 私聊点击返回后发送 `jarvis:conversation-history-changed` 事件；历史对话面板监听该事件和窗口 focus，自动刷新列表。
- 去掉私聊返回的重复 `onClose()` 调用，避免状态切换过快导致历史面板没来得及刷新。
- 修复 schedule_guard 的时区时间比较，避免 aware/naive datetime 混用。

## 验证

- 后端语法检查通过：`python -m py_compile app\api\v1\jarvis_router.py app\jarvis\tool_runtime.py app\jarvis\persistence.py`
- 前端类型检查通过：`npm.cmd run type-check`
