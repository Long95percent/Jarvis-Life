# Jarvis 长期记忆前端使用说明

## 你应该怎么体验这个功能

这次不是用 curl 或接口命令验证，而是在 Jarvis 产品界面里直接体验：和 Agent 私聊，让系统自动形成长期记忆，然后在右侧「长期记忆」面板查看、筛选、刷新和删除。

## 前端入口

打开 Jarvis 页面：

```text
/jarvis
```

右侧栏现在分成两块：

1. 「长期记忆」：显示系统已经记住的内容。
2. 「主动消息」：原来的主动消息列表。

## 推荐体验路径

1. 进入 `/jarvis`。
2. 选择 Mira 或 Maxwell 进入私聊。
3. 输入一条能产生长期记忆的话，例如：

```text
我最近备考雅思压力很大，别太频繁提醒我，我昨晚又熬夜了。
```

4. 等 Agent 回复完成。
5. 看右侧「长期记忆」面板。
6. 如果没有立刻出现，点击「刷新」。
7. 使用下拉筛选：
   - 全部
   - 偏好
   - 关心方式
   - 情绪
   - 作息
   - 长期目标
   - 关系
   - 事实
   - 约束
8. 找到不想保留的记忆，点击卡片右上角「删除」。
9. 确认删除后，这条记忆会从面板消失，后续也不会再注入私聊上下文。

## 面板上能看到什么

每条记忆卡片会显示：

- 记忆类型，例如「情绪」「作息」「长期目标」。
- 隐私等级，例如「普通」「隐私」「敏感」。
- 记忆内容。
- 来源 Agent。
- 上次被召回使用的时间。
- 删除按钮。

## 这个功能现在怎么工作

真实链路是：

1. 用户和 Agent 私聊。
2. Agent 回复完成后，后端用 LLM JSON 提取器提取长期记忆。
3. 如果 LLM 提取失败，会自动退回规则提取，不影响聊天。
4. 记忆写入 `jarvis_memories`。
5. 下一轮私聊开始前，系统会召回重要记忆并注入上下文。
6. 前端「长期记忆」面板读取这些记忆并允许删除。

## 判断功能好不好的观察点

体验时重点看这些点：

- 是否真的能从自然聊天里生成记忆，而不是必须填表。
- 生成的记忆是否太啰嗦、太像原文，还是足够概括。
- 情绪/作息类内容是否被标成「隐私」或「敏感」。
- 删除后是否不再出现在面板里。
- 下一轮聊天里，Agent 是否能自然参考这些记忆，但不会机械复读。

## 当前限制

- 面板需要手动点「刷新」，还没有聊天结束后自动刷新。
- 召回策略仍偏简单，主要按重要性和更新时间，不是完整语义检索。
- 还没有编辑记忆，只能删除。
- 还没有专门的趋势页，例如连续熬夜、连续焦虑的周趋势。
