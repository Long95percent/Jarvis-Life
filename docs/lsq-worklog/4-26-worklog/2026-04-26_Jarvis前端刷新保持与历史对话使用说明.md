# Jarvis 前端刷新保持与历史对话使用说明

## 这次修了什么

- 刷新浏览器后，不应该再回到主页。
- 当前正在看的私聊、圆桌或 Brainstorm 会尽量恢复。
- 右下角「主动消息」已替换为「历史对话」。
- 历史对话会落盘保存，用户可以自己删除。
- 创建超过 7 天且从未打开过的历史会自动清理。

## 怎么体验

1. 打开 `/jarvis`。
2. 进入一个 Agent 私聊，例如 Mira。
3. 发送一条消息。
4. 直接刷新浏览器。
5. 页面应继续停留在私聊界面，而不是回主页。
6. 看右下角「历史对话」，应能看到刚才的私聊记录。
7. 点击历史对话标题，可以恢复到对应对话。
8. 点击「删除」，可以删除这条历史。

## Brainstorm 怎么体验

1. 打开 `/jarvis`。
2. 启动 Brainstorm 场景。
3. 等它开始输出后刷新浏览器。
4. 页面应恢复到 Brainstorm / 圆桌界面。
5. 如果该 session 已经有后端 transcript，会展示已有讨论，而不是重新跑一遍 start。
6. 右下角「历史对话」里可以看到 Brainstorm 入口。

## 历史对话规则

- 私聊：发送消息后自动保存历史。
- 圆桌 / Brainstorm：启动时先保存历史入口。
- 用户点击历史时，会更新 `last_opened_at`。
- 如果一条历史创建后 7 天内从未被打开，会在列表查询时自动软删除。
- 用户手动删除也是软删除，删除后不再显示。

## 当前限制

- 历史对话暂不支持重命名。
- 主动消息模块暂时被历史对话替换，如果还需要主动消息，建议后续改成顶部通知或抽屉。
- Brainstorm 的完整 transcript 恢复依赖后端是否把每个事件写入 session turns；当前至少保证入口和页面状态不会丢。
