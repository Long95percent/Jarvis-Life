# Jarvis-Life 比赛演示 Demo 方案

## 1. 演示总目标

本次演示不要把 Jarvis-Life 讲成“AI 助手”或“桌面聊天工具”，而要讲成：

> **个人生活操作系统：把人的目标、日程、心理状态、长期偏好、现实环境和多智能体协作连接起来，让 AI 从回答问题进入决策、执行、反馈和记忆闭环。**

演示重点不是“它能聊天”，而是证明它能完成普通助手项目很难做到的四件事：

1. **理解人的状态**：压力、疲惫、睡眠、日程密度会影响决策。
2. **组织多角色判断**：不同智能体不是角色扮演，而是专业分工。
3. **把建议落到系统里**：日程、计划、关怀、记忆、主动消息都能形成闭环。
4. **越用越懂用户**：Shadow Learner 和长期记忆沉淀个人画像。

---

## 2. 建议演示顺序

如果现场时间允许，建议按这个顺序：

1. **30 秒定位**：说明它不是助手，而是个人生活操作系统。
2. **Demo 1 日程计划**：先展示最容易理解的“从一句话到日历落地”。
3. **Demo 2 心理 + 学习决策**：展示人的状态参与决策。
4. **Demo 3 圆桌协作**：展示多智能体不是摆设。
5. **Demo 4 关怀闭环**：展示长期陪伴和反馈调节。
6. **Demo 5 本地生活机会**：展示现实环境参与建议。
7. **Demo 6 个人画像**：用 Shadow Learner 收尾，强调越用越懂。

如果时间短，优先演示 Demo 1、Demo 2、Demo 6。

---

## 3. Demo 1：一个月奶茶计划 —— 从一句话目标到长期日程闭环

### 演示目标

展示 Jarvis-Life 能把一个自然语言长期安排，转成计划、日程、待确认动作和可追踪执行结果。

### 场景痛点

普通助手会说“你可以每周安排一次奶茶放松”，但不会真的帮用户检查时间、生成计划、写入日历，也不会处理周期性同名日程。

### 推荐输入

```text
接下来一个月，每周三和周六下午给我安排一次奶茶放松时间，每次 30 分钟，别和已有日程冲突。
```

如果想更稳，可以用更明确版本：

```text
从 5 月 6 日开始，一个月内每周三和周六 16:00 到 16:30 给我安排“奶茶放松”，如果当天已有冲突就帮我换到 17:00。
```

### 展示步骤

1. 打开 Jarvis 私聊，选择 Maxwell 或 Alfred。
2. 输入长期安排需求。
3. 展示聊天框中的真实执行步骤：确认智能体、读取上下文、判断工具、生成计划、保存记录。
4. 展示生成的待确认动作或长期计划。
5. 用户确认后，展示日历/计划中出现多个“奶茶放松”。
6. 强调系统现在允许不同日期同名日程，同一天同名才拦截。

### 可展示能力

- 自然语言日程理解。
- 长期/周期性计划拆解。
- 日程冲突意识。
- 待确认写入机制。
- 同名日程合理去重。
- 真实执行步骤流式展示。

### 差异化表达

> 普通助手只给建议，Jarvis-Life 把“想喝一个月奶茶”变成真实日历安排，并处理冲突、确认和后续追踪。

### 风险与兜底

- 如果模型没有自动拆出完整一个月计划，改用明确日期和时间输入。
- 如果现场日历数据混乱，先清理或换一个新 session。
- 如果确认动作未出现，展示真实步骤和返回内容，再说明这是可扩展到待确认动作的链路。

## 4. Demo 2：疲惫但要学习 —— 心理状态参与学习决策

### 演示目标

展示 Jarvis-Life 不只是学习规划工具，而是把学习策略、心理压力、日程安排和恢复节奏放在一起判断。

### 场景痛点

普通学习助手只会给学习计划，心理陪聊只会安慰，日历工具只会排时间。但真实问题是：用户现在是否还有能力继续学？应该硬撑、降低强度，还是调整到明天？

### 推荐输入

```text
我明天要考试，但我现在很焦虑、很累，还想硬撑两个小时学习。你帮我判断今晚到底该怎么安排。
```

### 展示步骤

1. 先在 Mira 或 Alfred 私聊输入这句话。
2. 展示系统对焦虑、疲惫、考试压力的回应。
3. 如果触发圆桌，进入“学习与恢复决策”场景。
4. 展示不同角色视角：
   - Mira：情绪和安全边界；
   - Athena：高收益低负担复习策略；
   - Maxwell：今晚和明早的时间安排；
   - Alfred：最终折中方案。
5. 最后展示可执行建议：低强度复习、睡前恢复、明早冲刺。

### 可展示能力

- 心理状态识别。
- 学习策略与认知负荷管理。
- 日程安排和恢复时间平衡。
- 多智能体协作或圆桌决策。
- 可转成计划或日程。

### 差异化表达

> 这不是学习助手，也不是心理陪聊，而是把“能不能学、怎么学、什么时候停、明天怎么接上”放进同一个生活决策里。

### 风险与兜底

- 避免输入极端心理风险文本，保持在焦虑、疲惫、压力范围。
- 如果圆桌慢，演示私聊结果即可，再展示圆桌入口和历史场景。
- 强调系统不是医疗诊断，而是轻量支持和生活决策辅助。

## 5. Demo 3：日程过载急救 —— 从混乱一天到可执行重排

### 演示目标

展示系统能处理真实生活中的多约束冲突：任务多、会议多、用户累、还想运动或恢复。

### 场景痛点

普通日历只告诉你“今天很满”，普通助手只说“注意休息”。Jarvis-Life 要判断哪些保留、哪些推迟、哪里插入恢复窗口。

### 推荐输入

```text
我今天事情太多了，下午还有会，晚上还想运动，但我已经有点撑不住了。帮我重新排一下今天，别把我压垮。
```

### 展示步骤

1. 打开首页 Dashboard，展示日程、状态、天气或本地生活卡片。
2. 进入 Alfred 或 Maxwell 私聊。
3. 输入过载场景。
4. 展示系统读取日程和上下文。
5. 展示建议：保留关键事项、推迟低优先级任务、安排恢复窗口、运动降强度。
6. 如出现待确认日程，展示确认后写入。

### 可展示能力

- 日程密度/压力意识。
- 日程读取与冲突判断。
- Maxwell 执行重排。
- Mira 恢复边界。
- Nora/Leo 对体力、饮食、轻活动的建议。
- 待确认动作写入。

### 差异化表达

> 普通工具看见的是时间表，Jarvis-Life 看见的是一个可能已经超载的人。

### 风险与兜底

- 演示前准备几条日程，避免空日历导致效果弱。
- 如果工具调用不稳定，可重点展示真实步骤和多因素建议。

## 6. Demo 4：心理关怀闭环 —— 从一次表达低落到后续反馈

### 演示目标

展示 Jarvis-Life 的心理模块不是一次性安慰，而是状态记录、趋势查看、主动关怀和反馈调节。

### 场景痛点

用户的压力和疲惫通常不会进入任务系统。普通助手可以安慰一次，但不会形成后续关怀，也不会根据用户反馈减少打扰。

### 推荐输入

```text
我最近睡不好，事情也很多，有点焦虑，不知道是不是自己太差劲了。
```

### 展示步骤

1. 选择 Mira 私聊。
2. 输入压力表达。
3. 展示 Mira 的轻量陪伴和安全边界。
4. 展示关怀卡片或主动消息。
5. 打开心理趋势/关怀面板。
6. 展示用户反馈：有帮助、稍后提醒、不需要、太频繁。

### 可展示能力

- 情绪和压力识别。
- 心理状态记录。
- Care trends / day detail。
- 主动关怀消息。
- 反馈闭环，避免过度打扰。
- 安全边界：不做医疗诊断。

### 差异化表达

> 普通助手会安慰一次，Jarvis-Life 会把这次状态变成后续生活决策和关怀节奏的一部分。

### 风险与兜底

- 避免输入自伤等高风险内容。
- 演示重点放在“轻量陪伴 + 趋势 + 反馈”，不要讲成医疗产品。

## 7. Demo 5：周末恢复圆桌 —— 不是推荐活动，而是生活决策

### 演示目标

展示本地生活、恢复需求、日程空档和多智能体圆桌如何共同参与一个生活决策。

### 场景痛点

普通推荐系统只问“你想去哪”，普通日历只知道“你有没有空”。用户真正需要的是：我这周很累，周末到底该出去、学习、社交，还是留白恢复？

### 推荐输入

```text
这周很累，周末我想恢复一下，但又不想完全躺平。帮我决定怎么过，最好能结合附近有什么轻松活动。
```

### 展示步骤

1. 从场景入口进入周末恢复/本地生活类圆桌。
2. 展示 Mira 判断恢复需求。
3. 展示 Leo 提供附近活动或生活机会。
4. 展示 Nora 评估体力和饮食节奏。
5. 展示 Maxwell 看周末时间安排。
6. Alfred 汇总出一个“半天活动 + 半天恢复”的方案。

### 可展示能力

- 多智能体圆桌舞台。
- 本地生活机会搜索与缓存。
- 天气、位置、时间上下文。
- 恢复需求和活动建议结合。
- Decision / Brainstorm 结构化结果。

### 差异化表达

> 普通推荐系统推荐地点，Jarvis-Life 先判断你现在需要什么样的生活安排，再决定是否推荐活动。

### 风险与兜底

- 本地生活搜索依赖外部搜索质量，建议提前准备城市和关键词。
- 如果外部搜索不稳定，可用已缓存结果或演示圆桌结构。

## 8. Demo 6：个人画像与偏好学习 —— AI 越用越懂你

### 演示目标

展示 Jarvis-Life 的长期价值：系统能通过旁路学习和长期记忆沉淀用户偏好，并在个人画像页展示。

### 场景痛点

普通 AI 工具每次对话都像重新开始。即使支持上下文，也很少把偏好变成可见、可管理、可用于后续决策的系统能力。

### 推荐输入

连续在私聊中输入：

```text
我不喜欢太频繁提醒，除非真的快迟到了。
我早上效率最高，难任务尽量放上午。
我压力大时希望你先帮我拆成三步，不要讲大道理。
晚上尽量不要安排高强度任务。
我喜欢直接给三个选项，不要太长。
```

### 展示步骤

1. 连续输入几条稳定偏好。
2. 等待后台 Shadow Learner 观察和提取。
3. 打开“个人画像”页。
4. 展示基础资料、系统学习到的偏好、长期记忆摘要。
5. 回到私聊，让系统按偏好给出更短、更低打扰的建议。

### 可展示能力

- Shadow Learner 旁路学习。
- 长期记忆。
- 个人画像页。
- 偏好参与后续对话。
- 用户可见和可管理的个性化。

### 差异化表达

> 普通助手靠 prompt 暂时记住你，Jarvis-Life 通过旁路学习和个人画像持续理解你。

### 风险与兜底

- 当前偏好提取可能每若干次观察才运行，演示前可提前准备数据。
- 如果现场没有立刻出现偏好，展示长期记忆摘要和接口能力。

## 9. 现场风险与兜底方案

### 9.1 模型或网络慢

- 优先演示私聊真实执行步骤，让等待过程可见。
- 减少圆桌轮数，只展示一轮和总结。
- 提前准备 prompt，减少现场输入时间。

### 9.2 外部检索不稳定

- 本地生活 demo 可以使用缓存结果。
- 不要把演示重点放在“搜索结果一定最新”，而是放在“外部信息能进入生活决策”。

### 9.3 心理模块输出不可控

- 不输入极端风险文本。
- 强调系统是轻量陪伴和生活节奏支持，不是医疗诊断。

### 9.4 日程数据混乱

- 演示前准备一个干净 session。
- 预置几条日程，确保日程过载 demo 有内容。
- 长期计划 demo 用明确日期和时间。

### 9.5 RAG 知识库当前不作为主 demo

- 当前 `/knowledge` 页面已经换成个人画像。
- RAG 能力可以作为技术储备提及，不建议作为现场主演示。

## 10. 30 秒 / 3 分钟 / 8 分钟演示版本

### 30 秒版本

> Jarvis-Life 不是一个 AI 助手，而是个人生活操作系统。它把日程、长期目标、心理状态、现实环境和长期记忆连接起来，让 AI 不只回答问题，而是参与判断、安排、执行和后续反馈。比如用户说“我很累但明天要考试”，系统会同时考虑学习策略、心理压力、日程安排和恢复节奏，而不是简单生成一份学习计划。

### 3 分钟版本

1. 用一句话定位开场。
2. 演示 Demo 1：一句话生成一个月奶茶计划。
3. 快速展示真实执行步骤和日历落地。
4. 演示 Demo 6：个人画像，说明系统会长期学习偏好。
5. 结束时强调：普通助手回答建议，Jarvis-Life 形成闭环。

### 8 分钟版本

1. 30 秒定位。
2. Demo 1：长期日程计划。
3. Demo 2：疲惫学习决策。
4. Demo 4：心理关怀闭环。
5. Demo 5：周末恢复圆桌。
6. Demo 6：个人画像与长期记忆。
7. 总结差异：状态感知、多智能体协作、行动落地、旁路学习。

## 11. 最推荐的比赛主线

如果只能讲一条主线，建议用：

```text
用户不是机器，所以生活系统不能只看任务。
Jarvis-Life 把目标、日程、心理状态、现实环境和长期偏好接成闭环，
让 AI 从回答问题升级为个人生活操作系统。
```

这条主线能同时覆盖日程、心理、圆桌、记忆和执行落地，最能体现项目和普通桌面助手的差异。
