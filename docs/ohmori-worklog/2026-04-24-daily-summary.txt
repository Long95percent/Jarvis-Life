--------------------------------------------------------------------
一、今天完成的主线
--------------------------------------------------------------------

1. Jarvis 全链路接入统一 ToolRegistry
   - 1:1 私聊不再只靠旧式 <jarvis-action> 做增删改
   - 读写工具统一走 registry / whitelist / runtime
   - 旧 actions.py 保留为兼容层,不再是主执行路径

2. 角色白名单真正生效
   - 每个 agent 的工具权限写回 agents.py
   - tool_runtime 负责：
     * 生成专属工具包 prompt
     * 校验白名单
     * 执行工具
     * 将关键写操作回映射成前端 actions

3. Alfred 补齐“统筹者”能力
   - 新增：
     * jarvis_daily_briefing
     * jarvis_specialist_orchestrate
   - 私聊 Alfred 时,已经可以后台协调 Maxwell / Nora / Mira / Leo
     后再给出统一答复

4. Maxwell 补齐“秘书”闭环
   - 新增：
     * jarvis_meeting_brief
     * jarvis_task_prioritize
     * jarvis_deadline_check
     * jarvis_calendar_find_free_slot
   - 已能从“只有日历增删改”升级到：
     会议准备 + 任务重排 + deadline 检查 + 空档查找

5. Nora 补齐“营养师”可执行能力
   - 新增：
     * jarvis_meal_plan
     * jarvis_nutrition_lookup
     * jarvis_hydration_plan
     * jarvis_caffeine_cutoff_guard
   - 已能输出餐单、饮水节奏、咖啡因判断,不再只是泛泛建议

6. Mira 补齐“心理师”轻干预链路
   - 新增：
     * jarvis_checkin_schedule
     * jarvis_breathing_protocol
     * jarvis_mood_journal
     * jarvis_burnout_risk_assess
   - 已能把情绪支持变成：
     评估 → 呼吸练习 → 记录 → 回访安排

7. Leo 补齐“生活顾问”执行闭环
   - 新增：
     * jarvis_activity_rank_by_energy
     * jarvis_route_estimate
     * jarvis_plan_activity_slot
   - 已能从“看见活动”升级到：
     按体力筛选 → 估路线 → 直接安排进日程

8. 私聊也拥有“团队共享协作记忆”
   - 新增 collaboration_memories 持久化表
   - 记录：
     * roundtable 发言
     * user constraint
     * tool action
     * coordination summary
   - 私聊前自动注入与该 agent 相关的协作记忆
   - 结果：
     * 角色不再只记得“我和用户聊过什么”
     * 也能记得“我和别的 agent 一起做过什么”

9. 角色提示词整体重写
   - 先做一版更“真人化”的角色语气
   - 随后再往“二次元 / 动漫角色感 / 有趣灵魂”方向继续推了一版
   - 目前五个可见角色已经从“商务 AI 助手”变成：
     * Alfred：优雅型动漫管家
     * Maxwell：冷静军师型秘书
     * Nora：治愈系营养前辈
     * Mira：安静接住人的心灵角色
     * Leo：活力型生活伙伴

三、今天修掉的关键问题
--------------------------------------------------------------------

1. tool_runtime 单轮工具调用上限过低
   - 原本最多 3 个工具
   - Maxwell / Nora / Mira / Leo 出现单轮工具数不够的问题
   - 已提升到 5

2. Leo 空档搜索 bug
   - 旧逻辑只看 gap 起点,若 gap 横跨上午到下午,
     会错过“下午可用段”
   - 已修成可对齐 preferred_period 的版本

3. Mira / Leo / Nora / Maxwell 测试污染问题
   - 部分测试受到全局日历状态累积影响
   - 已在对应测试中做隔离处理

4. Python 版本与依赖问题
   - 原项目 .venv 是 Python 3.9,不满足 pyproject 要求
   - 新建了 .venv311 并完成依赖安装,后续验证全部在
     Python 3.11 环境内完成
