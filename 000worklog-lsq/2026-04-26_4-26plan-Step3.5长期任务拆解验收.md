# 验收说明：Step 3.5 Maxwell 长短期任务拆解与后台任务清单

创建时间：2026-04-26（Asia/Shanghai）

## 1. 本次变更范围

本次已从计划进入功能落地，目标是让 Maxwell 具备“长期/短期/未来任务拆解”能力，并通过待确认卡片把计划保存进后台任务清单。

已落地内容：

1. 新增后台任务清单持久化表 `background_tasks`，用于保存长期任务、未来任务、周期计划和项目型目标。
2. 新增 Maxwell 专属任务拆解工具 `jarvis_task_plan_decompose`，用于把“我要考雅思”“暑假后去旅行”等自然语言需求拆成里程碑、子任务和可进入日历的候选安排。
3. Maxwell 已加入该工具白名单，并在系统提示中明确长期目标不要直接塞满日历，而是先生成后台任务计划。
4. 工具运行时支持 `task.plan` 类型确认动作，先生成计划草案，再等待用户确认。
5. 确认 `task.plan` 后，后端把计划写入后台任务清单。
6. Java Gateway 已代理后台任务查询接口。
7. 前端私聊面板已支持显示“待确认长期/后台任务计划”卡片，并可确认加入后台任务清单或取消。
8. 非 Maxwell 智能体收到日程/任务类需求时，增加初步路由到 Maxwell 的判断逻辑。

## 2. 涉及文件

- `shadowlink-ai/app/jarvis/persistence.py`：新增后台任务表和保存/读取/列表方法。
- `shadowlink-ai/app/tools/jarvis_tools.py`：新增 `JarvisTaskPlanDecomposeTool` 和输入 schema。
- `shadowlink-ai/app/core/lifespan.py`：注册 `jarvis_task_plan_decompose` 内置工具。
- `shadowlink-ai/app/jarvis/agents.py`：为 Maxwell 开启工具白名单，并强化长期任务规划提示。
- `shadowlink-ai/app/jarvis/tool_runtime.py`：支持 `task.plan` 待确认动作。
- `shadowlink-ai/app/api/v1/jarvis_router.py`：支持确认后台任务计划、查询后台任务、非 Maxwell 路由判断。
- `shadowlink-server/shadowlink-gateway/src/main/java/com/shadowlink/gateway/controller/JarvisProxyController.java`：代理 `/background-tasks`。
- `shadowlink-web/src/services/jarvisApi.ts`：新增 `BackgroundTask` 类型与后台任务查询 API，放宽确认动作返回类型。
- `shadowlink-web/src/components/jarvis/AgentChatPanel.tsx`：新增 `task.plan` 确认卡片。

## 3. 如何验收

### 验收项 A：Maxwell 直接处理长期任务

在前端私聊 Maxwell，输入：

```text
我想准备雅思，目标是一个月后开始系统学习，你帮我拆一下长期计划
```

预期结果：

- Maxwell 不应只回复一段普通文本。
- 页面应出现“待确认长期/后台任务计划”卡片。
- 卡片中应包含计划标题、类型、里程碑或秘书还需要确认的问题。
- 点击“确认加入后台任务”后，卡片状态变为“已加入后台任务清单”。

### 验收项 B：其它 Agent 触发后路由到 Maxwell

在前端私聊 Mira/Nora/Leo 等非 Maxwell 智能体，输入：

```text
我暑假后想去日本玩，你帮我先规划一下准备事项
```

预期结果：

- 系统应把任务规划需求交给 Maxwell 处理。
- 不应出现“tool not allowed for mira”或普通 Agent 无权限调用秘书工具的错误。
- 最终仍应出现 `task.plan` 待确认卡片。

### 验收项 C：确认后进入后台任务清单

点击确认后，访问接口：

```text
GET /api/v1/jarvis/background-tasks
```

预期结果：

- 返回数组中出现刚确认的任务。
- 任务应包含 `title`、`task_type`、`milestones`、`subtasks`、`calendar_candidates`、`original_user_request` 等字段。

如果通过 Java Gateway 访问，则检查：

```text
GET /api/jarvis/background-tasks
```

### 验收项 D：不要把长期计划直接塞满日历

用雅思、旅行、健身、长期学习等案例测试。

预期结果：

- 长期任务先进入后台任务清单。
- 日历只应在后续确认近期待办/时间块时写入，不应一次性生成几十条未来日程。

## 5. 通过标准

满足以下条件即通过：

- Maxwell 直接收到长期/未来任务时，能生成 `task.plan` 卡片。
- 非 Maxwell Agent 收到明显任务/日程规划需求时，能路由给 Maxwell。
- 用户确认后，任务能持久化进入 `background_tasks`。
- 前端不再把长期任务只显示为普通聊天文本。
- 后台任务不会绕过用户确认直接写入。
- 长期计划不会一次性塞满日历。

## 6. 已知边界

- 当前 `task.plan` 卡片支持确认/取消，但开始日期、结束日期、频率、单次时长的可视化编辑还未完全落地。
- 当前后台任务清单已有后端接口，但还没有独立的前端列表管理页。
- 路由到 Maxwell 的判断已做基础关键词规则，后续应升级为统一 `Intent Router + Maxwell Task Planning Skill`。
