# 2026-04-26 Step3.6 记忆提取层 Demo Run / Demo Trace 路演脚本

## 路演目标

证明系统已经具备“测试画像重置 → 加载固定画像 → 执行路演步骤 → 记录记忆与追踪 → 导出 Demo Trace”的最小闭环。重点不是展示完整自动记忆智能，而是证明每一步可追溯、可复盘、可重复测试。

## 预设画像

使用 `worklog-lsq/test/2026-04-26_路演用户画像种子.md` 中的雅思备考、生活节律、暑假旅行组合画像。

建议 demo_run_id：`demo-20260426-ielts-rhythm-travel-001`

## 步骤 1：清空测试记忆

接口：

```http
POST /api/v1/jarvis/demo/reset
```

预期：

- 返回被清理的 `demo_runs`、`trace_events`、`memory_items` 数量。
- 普通会话、日程、后台任务不被清空。

讲解词：

> 这里先把路演专用的测试画像和测试记忆清空，保证每次演示从同一个起点开始，不受历史脏数据影响。

## 步骤 2：开始 Demo Run 并加载画像

接口：

```http
POST /api/v1/jarvis/demo/runs
Content-Type: application/json

{
  "demo_run_id": "demo-20260426-ielts-rhythm-travel-001",
  "seed_name": "ielts-rhythm-travel",
  "profile_seed": {
    "goals": ["IELTS 7.0", "暑假旅行准备"],
    "rhythm": {"sleep_issue": "最近熬夜", "preferred_care": "低打扰"},
    "care_preference": "先共情，再给少量可执行建议"
  },
  "reset_existing": false
}
```

预期：

- 返回 `demo_run`。
- `profile_seed` 原样保存，后续 Trace 能追溯演示上下文。

讲解词：

> 这一步相当于加载一个固定用户画像。以后每次路演都可以用同一个 seed 重放，方便比较模型和产品改动前后的表现。

## 步骤 3：记录一次用户输入与 Agent 回复

接口：

```http
POST /api/v1/jarvis/demo/runs/demo-20260426-ielts-rhythm-travel-001/trace
Content-Type: application/json

{
  "demo_step_id": "step-01-care-boundary",
  "event_type": "agent_reply",
  "agent_id": "mira",
  "user_input": "我最近备考雅思压力很大，但不要太频繁提醒我。",
  "agent_reply": "我会降低打扰频率，先帮你保留一个轻量复习节奏。",
  "tool_calls": [],
  "memory_events": [],
  "confirmation": {"required": false},
  "payload": {"scenario": "心理关心偏好"}
}
```

预期：

- 返回 `trace_event`。
- 包含 `demo_step_id`、输入、回复、确认状态和 payload。

讲解词：

> 这里先不强行写长期记忆，而是把本轮输入、回复、工具调用和确认动作记录到 Trace。之后任何路演结果都能回放。

## 步骤 4：保存一条敏感偏好记忆

接口：

```http
POST /api/v1/jarvis/demo/runs/demo-20260426-ielts-rhythm-travel-001/memories
Content-Type: application/json

{
  "demo_step_id": "step-01-care-boundary",
  "memory_kind": "care_preference",
  "content": "用户备考压力大时偏好低打扰关心，回复应先共情再给少量建议。",
  "source_text": "我最近备考雅思压力很大，但不要太频繁提醒我。",
  "sensitivity": "private",
  "confidence": 0.8,
  "importance": 0.9
}
```

预期：

- 返回 `memory_item`。
- 记忆带有类型、来源、敏感等级、置信度和重要性。

讲解词：

> 这就是 Step 3.6 要补的核心：不是保存所有聊天原文，而是把值得记的信息结构化，并且标记隐私等级和置信度。

## 步骤 5：导出 Demo Trace

接口：

```http
GET /api/v1/jarvis/demo/runs/demo-20260426-ielts-rhythm-travel-001/trace
```

预期：

- 返回 `demo_run`、`trace_events`、`memory_items`。
- 能看到画像 seed、每一步输入/回复、确认状态和写入的记忆。

讲解词：

> 最后导出 Demo Trace。这样每个路演步骤都能追溯到输入、Agent 回复、工具调用、确认动作和记忆写入结果，方便复盘和调试。

## 验收口径

- 能一键清空路演测试记忆。
- 能加载固定画像种子并开始 Demo Run。
- 能按 demo_step_id 记录输入、回复、工具调用、确认动作。
- 能保存带隐私等级、置信度、重要性的记忆条目。
- 能导出完整 Demo Trace。
