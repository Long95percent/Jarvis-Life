# Jarvis 长期记忆查看与删除说明

## 适用范围

本文档说明 Step 3.6 真实私聊记忆闭环落地后的记忆管理方式。当前记忆来自 `/api/v1/jarvis/chat` 私聊结束后的自动提取，保存到 `jarvis_memories`，并会在下一轮私聊前作为长期记忆注入上下文。

## 当前已支持的记忆类型

- `fact`：事实信息。
- `relationship`：人际关系、实体关系。
- `preference`：用户偏好。
- `constraint`：用户约束或明确要求。
- `long_term_goal`：长期目标、考试、旅行、项目等。
- `mood_signal`：情绪/压力信号，默认隐私级别较高。
- `rhythm_signal`：睡眠、熬夜、作息等生活节律信号。
- `care_preference`：关心方式偏好，例如低打扰、先共情再建议。

## 查看所有记忆

接口：

```http
GET /api/v1/jarvis/memories
```

示例：

```bash
curl http://127.0.0.1:8000/api/v1/jarvis/memories
```

返回结构示例：

```json
{
  "memories": [
    {
      "id": 12,
      "memory_kind": "care_preference",
      "content": "用户备考压力大时偏好低打扰关心。",
      "source_agent": "mira",
      "session_id": "chat-001",
      "source_text": "我最近备考雅思压力很大，别太频繁提醒我。",
      "structured_payload": {
        "source": "llm_memory_extractor"
      },
      "sensitivity": "private",
      "confidence": 0.82,
      "importance": 0.86,
      "created_at": 1770000000.0,
      "updated_at": 1770000000.0,
      "last_used_at": null,
      "status": "active"
    }
  ]
}
```

## 按类型筛选记忆

接口：

```http
GET /api/v1/jarvis/memories?memory_kind=care_preference
```

常用筛选：

```bash
curl "http://127.0.0.1:8000/api/v1/jarvis/memories?memory_kind=mood_signal"
curl "http://127.0.0.1:8000/api/v1/jarvis/memories?memory_kind=rhythm_signal"
curl "http://127.0.0.1:8000/api/v1/jarvis/memories?memory_kind=long_term_goal"
```

## 限制返回数量

接口：

```http
GET /api/v1/jarvis/memories?limit=20
```

示例：

```bash
curl "http://127.0.0.1:8000/api/v1/jarvis/memories?limit=20"
```

## 删除一条记忆

接口：

```http
DELETE /api/v1/jarvis/memories/{memory_id}
```

示例：

```bash
curl -X DELETE http://127.0.0.1:8000/api/v1/jarvis/memories/12
```

成功返回：

```json
{
  "deleted": true,
  "memory_id": 12
}
```

说明：当前删除是软删除，会把 `status` 标记为 `deleted`。默认查看接口只返回 `active` 记忆，因此删除后不会再被召回注入私聊上下文。

## 推荐使用流程

1. 先和某个 Agent 私聊，例如告诉 Mira：“我最近备考雅思压力很大，别太频繁提醒我。”
2. 调用 `GET /api/v1/jarvis/memories` 查看是否生成了 `care_preference`、`mood_signal`、`long_term_goal` 等记忆。
3. 如果某条记忆不准确，记录它的 `id`。
4. 调用 `DELETE /api/v1/jarvis/memories/{id}` 删除该记忆。
5. 再次调用 `GET /api/v1/jarvis/memories` 确认它不再返回。

## 当前注意事项

- 记忆提取目前是 LLM JSON 提取 + 规则兜底：LLM 失败时不会影响主对话，会自动退回规则提取。
- 心理、睡眠、压力类记忆会标记为 `private` 或 `sensitive`，但目前还没有前端权限分层页面。
- 召回策略目前按重要性和更新时间排序，还没有按当前问题做向量语义检索。
- 删除接口已可用，但还没有前端按钮，需要后续接入记忆管理页。
