# 2026-04-26 任务确认落盘与每日多日程显示修复 Worklog

## 用户反馈

- 点击“确认加入后台任务”后，必须确保真的保存并落盘，后续打开还能看到；如果失败要明确报错。
- 首页每日日程面板现在只显示最新添加的一条日程，这是严重问题；应该显示当天所有日程，显示不下就滚动。
- 长周期任务不应该每次都要求确认才能挂到后台；具体日历写入仍可确认。
- 秘书在添加日程前需要判断时间是否合理、是否冲突、是否建议取消/移动低优先级行程。
- 设置里需要能给每个角色配置 skill，并支持上传 skill 文件。

## 本次已修复

### 任务确认保存闭环

- `task.plan` 确认后调用 `save_background_task`。
- 保存后立即调用 `get_background_task` 回读校验。
- 如果回读失败，后端返回 500，并提示 `background_tasks` 写入问题。
- 前端确认卡成功后显示：`已保存到后台任务清单：xxx`。

### 每日日程显示全部

- 修复新增日程后 LifeContext 只写入 `[event]` 的问题。
- 新增日程后改为回写未来 24 小时 `get_upcoming_events(hours_ahead=24)` 的全部事件。
- 首页今日日程卡片移除 `slice(0, 5)`，改为展示全部并使用滚动条。

## 验证

- 前端类型检查：`npm.cmd run type-check` 通过。
- 后台任务保存/回读 smoke 通过。

## 仍需后续拆分实现

1. 长周期任务自动后台落盘，不每次确认。
2. 日程写入前冲突检查和重要性判断 Skill。
3. 设置页按角色配置 Skill。
4. Skill 文件上传、解析、绑定角色、权限确认策略。
