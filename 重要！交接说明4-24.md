# Jarvis-Life 项目交接说明

> 撰写日期：2026-04-23

---

## 一、项目简介

**Jarvis-Life**（代号 ShadowLink）是一个面向具体工作场景的 AI 工作台平台，核心能力包括：多策略 Agent 自动路由、本地 RAG 知识库检索、工具调用执行、以及流式对话交互。

---

## 二、启动说明

### 最快启动方式（Windows）

直接双击根目录的 `start.bat`，脚本会：

1. 自动检测 Python 环境（conda / venv）
2. 安装依赖（首次运行）
3. 启动 Python AI 服务（端口 8000）
4. 启动 React 前端（端口 3000）
5. 自动打开浏览器访问 `http://localhost:3000`

### 手动启动（分项启动）

**启动 Python AI 服务：**

```bash
cd shadowlink-ai
pip install -e ".[dev]"
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**启动 React 前端（新终端）：**

```bash
cd shadowlink-web
npm install
npm run dev
# 访问 http://localhost:3000
```

**启动 Java 后端（可选，功能完整版需要）：**

```bash
cd shadowlink-server
修复了一个bug:
在pom.xml增加了一个带版本说明的依赖
然后运行
mvnw.cmd clean install
mvnw.cmd spring-boot:run -pl shadowlink-starter


windows .cmd脚本
mvnw.cmd spring-boot:run -pl shadowlink-starter

Linux/macos
./mvnw spring-boot:run -pl shadowlink-starter
# 端口 8080
```

### Docker Compose 启动

```bash
cd deploy
cp .env.example .env       # 编辑 .env，填写 LLM_API_KEY 等
docker compose up -d --build
```

### 各服务端口

| 服务 | 端口 | 访问地址 |
|------|------|---------|
| React 前端 | 3000 | http://localhost:3000 |
| Python AI 服务 | 8000 | http://localhost:8000 |
| AI API 文档（Swagger） | 8000 | http://localhost:8000/docs |
| Java 网关 | 8080 | http://localhost:8080 |
| MySQL | 3306 | localhost:3306 |
| Redis | 6379 | localhost:6379 |
| gRPC | 50051 | localhost:50051 |

### 关键配置

LLM 提供商配置在根目录的 `llm_config.json`（该文件含 API Key，已被 .gitignore，需手动创建）。参考 `config/templates/llm_config.example.json`：

```json
{
  "active_agent_id": "deepseek",
  "agents": [
    {
      "id": "deepseek",
      "name": "DeepSeek (DashScope)",
      "type": "api",
      "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1",
      "model": "deepseek-v3.2",
      "api_key": "YOUR_KEY_HERE"
    }
  ]
}
```

## 三、项目结构总览

```
Jarvis-Life/
├── shadowlink-ai/          # Python AI 服务（核心）
├── shadowlink-server/      # Java API 网关（Spring Boot）
├── shadowlink-web/         # React 前端
├── shadowlink-electron/    # Electron 桌面壳（可选）
├── proto/                  # gRPC 协议定义
├── deploy/                 # Docker / Nginx / 数据库部署配置
├── docs/                   # 文档
├── config/                 # 配置模板
├── data/                   # 运行时数据（记忆、上传文件、向量索引）
├── llm_config.json         # LLM 提供商配置（含 Key，不入库）
├── tasks_config.json       # 任务管理配置
├── start.bat               # Windows 一键启动脚本
└── Makefile                # 统一构建命令
```

整体为三层架构：

```
React 前端（3000）
    ↕ HTTP / SSE / WebSocket
Java 网关（8080）
    ↕ HTTP REST + gRPC
Python AI 服务（8000 / 50051）
    ↕ SQL / HTTP
MySQL · Redis · Ollama（可选）
```

## 四、各模块职责

### `shadowlink-ai/` — Python AI 服务（核心）

项目最核心的部分，基于 FastAPI，负责所有 AI 逻辑。

| 子目录 | 职责 |
|--------|------|
| `app/agent/` | Agent 执行引擎：任务复杂度路由、ReAct、Plan-and-Execute、MultiAgent 编排 |
| `app/rag/` | RAG 检索增强管道：分块、向量嵌入、FAISS 索引、混合检索、重排序 |
| `app/llm/` | 统一 LLM 客户端：多提供商适配（OpenAI / DeepSeek / Anthropic / Ollama） |
| `app/tools/` | 工具集：代码执行、知识库搜索、网页搜索、文件操作、系统命令 |
| `app/mcp/` | Model Context Protocol 实现，工具互操作 |
| `app/file_processing/` | 文件解析：PDF、Word、Excel、PPT、Markdown、代码、图片（OCR） |
| `app/memory/` | 记忆系统：短期（会话内）、长期（跨会话）、情节（事件）、语义（知识图谱） |
| `app/api/v1/` | REST API 路由层：agent、rag、mcp、file、settings、health 等端点 |
| `app/api/grpc/` | gRPC 服务实现（端口 50051），供 Java 层调用 |
| `app/core/` | 基础设施：日志、异常处理、中间件、依赖注入、生命周期管理 |
| `app/jarvis/` | Jarvis 人格层：个性定义、主动建议、上下文理解 |
| `app/plugins/` | 插件系统：插件加载器、内置插件（计算器、翻译） |
| `app/config.py` | 全局配置管理（Pydantic Settings，环境变量前缀 `SHADOWLINK_*`） |

**Agent 执行策略路由逻辑：**

| 策略 | 触发条件 | 特点 |
|------|---------|------|
| Direct | 简单问答、翻译、改写 | 单次 LLM 调用 |
| ReAct | 信息检索、需调用少量工具 | 推理→行动→观察循环 |
| Plan-and-Execute | 复杂任务、多步分解 | 先规划再逐步执行，支持重规划 |
| Supervisor | 多领域协调（暂为备用） | 多 Agent 编排 |

---

### `shadowlink-server/` — Java API 网关

基于 Spring Boot 3.2，Monorepo 结构，分 8 个模块：

| 模块 | 职责 |
|------|------|
| `shadowlink-common` | 公共 DTO、工具类、常量 |
| `shadowlink-auth` | JWT 认证授权（登录、刷新 Token、鉴权过滤器） |
| `shadowlink-business` | 工作模式（WorkMode）、会话、对话、知识库的 CRUD 业务逻辑 |
| `shadowlink-ai-bridge` | AI 服务网桥：通过 HTTP + gRPC 转发请求到 Python |
| `shadowlink-session` | 会话存储与管理（配合 Redis） |
| `shadowlink-websocket` | WebSocket 实时通信（推送 Agent 执行事件到前端） |
| `shadowlink-gateway` | API 路由层、请求拦截、过滤 |
| `shadowlink-starter` | Spring Boot 应用入口，整合所有模块，含 application.yml 和数据库初始化脚本 |

---

### `shadowlink-web/` — React 前端

基于 React 18 + Vite + TypeScript + Tailwind CSS。

| 目录 | 职责 |
|------|------|
| `src/pages/` | 三个主页面：Jarvis 工作台（主页）、知识库管理、设置 |
| `src/components/jarvis/` | Jarvis 首页、Agent 聊天面板、圆桌舞台、信息卡片、主动消息流 |
| `src/components/agent/` | Agent 执行过程展示：思维过程、工具调用卡片、计划进度 |
| `src/components/ambient/` | 环境氛围组件：背景效果、模式切换器、模式设置弹窗 |
| `src/services/` | API 客户端：Axios 实例、SSE 事件流处理、WebSocket 连接 |
| `src/stores/` | Zustand 状态管理：Agent 状态、聊天状态、氛围模式状态 |
| `src/hooks/` | 自定义 Hook：useAgent、useSSE、useWebSocket |
| `src/theme/` | 主题系统（暗色/亮色）和颜色定义 |

---

### `shadowlink-electron/` — Electron 桌面壳

将 React 前端打包为桌面应用，主进程在 `src/main.ts`，使用 Electron Builder 打包发行。目前为可选模块，日常开发以 Web 模式为主。

---

### `proto/` — gRPC 协议定义

定义 Java ↔ Python 之间的 gRPC 接口契约：

| 文件 | 服务 |
|------|------|
| `agent_service.proto` | Agent 聊天、流式对话、任务分类 |
| `rag_service.proto` | RAG 查询、文档摄入 |
| `mcp_service.proto` | MCP 工具执行 |
| `memory_service.proto` | 记忆读写 |

运行 `generate.sh` 或 `make proto` 可重新生成各语言代码。

---

### `deploy/` — 部署配置

| 文件/目录 | 职责 |
|----------|------|
| `docker-compose.yml` | 生产环境完整编排 |
| `docker-compose.dev.yml` | 开发环境叠加配置（H2 内存数据库可选） |
| `.env` / `.env.example` | 环境变量（数据库密码、API Key 等） |
| `scripts/init-db.sql` | 数据库初始化 DDL |
| `nginx/` | Nginx 反向代理配置 |
| `mysql/` | MySQL 自定义配置 |
| `redis/` | Redis 自定义配置 |

## 五、目前已有的功能

### 已实现并可用

- **多策略 Agent**：Direct / ReAct / Plan-and-Execute 三种策略，根据任务复杂度自动路由
- **流式输出**：SSE 流式返回 Agent 思考过程、工具调用、执行结果（思考/行动/观察/Token 逐帧推送）
- **RAG 知识库**：支持文档上传（PDF / Word / Excel / PPT / Markdown / 代码），建立向量索引，混合检索+重排序
- **工具调用**：代码执行（Python/SQL）、网页搜索、本地文件搜索、知识库搜索、文件操作
- **多 LLM 提供商**：OpenAI / DeepSeek（阿里云 DashScope）/ Anthropic Claude / Ollama 本地模型，运行时切换
- **MCP 协议**：Model Context Protocol 工具互操作
- **工作模式管理**：通过 Java 侧 CRUD API 管理工作模式（系统提示词、推荐策略、工具白名单、知识库作用域）
- **会话管理**：创建/删除会话，保存对话历史
- **JWT 认证**：登录、Token 刷新、鉴权过滤
- **短期记忆**：当前会话内的消息缓冲上下文
- **文件处理**：PDF、Word、Excel、PPT、Markdown、代码、图片（OCR）解析
- **Jarvis 前端工作台**：Jarvis 主页、Agent 聊天面板、思维过程可视化、工具调用展示、计划进度展示
- **氛围模式**：前端支持多种环境氛围切换（背景效果 + 模式配置）
- **Docker 一键部署**：完整 docker-compose 配置，含 MySQL、Redis、Nginx

### 已搭架子、暂未完全启用

- **MultiAgent 编排**：Supervisor / Swarm / 层次编排代码已有，非当前核心路径
- **长期记忆**：代码框架已有，跨会话持久化能力待完善
- **语义记忆（知识图谱）**：框架存在，未深度集成
- **插件系统**：插件加载器和内置插件（计算器、翻译）已有，扩展接口待完善
- **gRPC 全链路**：proto 定义和生成代码已就绪，部分 servicer 实现待补充
- **Prometheus 监控**：deploy/prometheus 配置已有，未完整接入

### 规划中、尚未开始

- 企业级权限管理
- 插件市场
- 知识库版本控制
- 完整审计日志
- 自定义 Agent 市场
